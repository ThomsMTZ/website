# Internationalization (I18N) System

This project uses a custom I18N system to support multiple languages (currently English and French).

## Structure

```
utils/i18n/
├── I18nContext.jsx     # React Context Provider for language management
├── en.js              # English translations
├── fr.js              # French translations
├── translations.js    # Exports all translations
└── index.js          # Main export file
```

## Usage

### In Components

```jsx
"use client";
import { useTranslation } from "@/utils/i18n";

function MyComponent() {
  const { t, language, changeLanguage } = useTranslation();
  
  return (
    <div>
      <h1>{t.navbar.about}</h1>
      <button onClick={() => changeLanguage('fr')}>
        Switch to French
      </button>
    </div>
  );
}
```

### Available Translation Keys

The translations are organized by component/section:

- `navbar` - Navigation menu items
- `hero` - Hero section text and buttons
- `about` - About section
- `skills` - Skills section
- `experience` - Experience section  
- `projects` - Projects section
- `education` - Education section
- `contact` - Contact section and form
- `blog` - Blog section
- `footer` - Footer text
- `notFound` - 404 page

## Adding New Languages

1. Create a new translation file (e.g., `es.js` for Spanish):

```javascript
export const es = {
  navbar: {
    about: "ACERCA DE",
    // ... other translations
  },
  // ... other sections
};
```

2. Update `translations.js`:

```javascript
import { en } from './en';
import { fr } from './fr';
import { es } from './es';

export const translations = {
  en,
  fr,
  es,
};

export const supportedLanguages = ['en', 'fr', 'es'];
```

## Adding New Translation Keys

When adding new text to components:

1. Add the key to **all** language files (en.js, fr.js, etc.)
2. Use the translation in your component with `t.section.key`

Example:

```javascript
// In utils/i18n/en.js
export const en = {
  // ... existing keys
  newSection: {
    title: "New Title",
    description: "New Description"
  }
};

// In utils/i18n/fr.js
export const fr = {
  // ... existing keys
  newSection: {
    title: "Nouveau Titre",
    description: "Nouvelle Description"
  }
};

// In your component
function NewComponent() {
  const { t } = useTranslation();
  return (
    <div>
      <h1>{t.newSection.title}</h1>
      <p>{t.newSection.description}</p>
    </div>
  );
}
```

## Language Persistence

The selected language is automatically saved to `localStorage` and restored on page reload.

## Language Switcher

A language switcher component is included in the navigation bar that allows users to toggle between available languages. You can also create custom language switchers using the `changeLanguage` function from the `useTranslation` hook.
